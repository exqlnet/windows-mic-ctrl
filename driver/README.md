# Windows 虚拟麦驱动骨架

> 目标：提供应用自建虚拟麦克风端点（可被 QQ 识别）。

当前阶段仅完成工程骨架与接口协议定义，尚未实现可安装的内核态虚拟麦驱动。

## 目录约定

- `driver/protocol/`：用户态与驱动态协议（IOCTL / 共享内存）

## 计划分层

1. 驱动态端点层：暴露 Capture Endpoint（虚拟麦）
2. 用户态桥接层：采集物理麦并写入驱动共享缓冲
3. 控制层：门控状态、快捷键状态同步
4. 安装层：签名、安装、卸载、回滚
